SkillUpTn - Documentation complete et design clair

Reference fonctionnelle et technique ultra detaillee: interface, backend, IA, flux de validation RH/manager/employe, gestion des refus, penalties, notification, historique et qualite de donnees.

React + TypeScriptNestJSMongoDBJWT + RBACWebSocketNLP/AI

1) Architecture globale

Frontend
`frontend/src`
Pages par role, DataContext, fetchWithAuth, toasts globaux, accessibilite, animations.
Backend
`backend/src`
Modules: auth, users, activities, recommendations, notifications, manager, chat, audit.
AI Service
Moteur de parsing prompt, matching skills, scoring multi-criteres, retraining calibrateur.

Flux central

Creation activite -> prompt RH -> generate recommendations -> tri score_total desc -> validation RH -> validation manager -> employe accepte/refuse -> mise a jour competences -> activity_history + notification + audit -> retraining periodique

3) Pages frontend et boutons (en detail)

HRRecommendations

  • Lancer recommandation: lance IA sur prompt.
  • Envoyer au manager: pousse decisions RH.
  • Confirmer activite: confirmation explicite RH.
  • Popup places manquantes: annuler ou confirmer quand meme.
  • Refus employe: clic employe -> affiche cause -> choix diminuer/garder.
  • Ajouter a la base: creation competence manquante (niveau 1..5).

HRActivities

  • Liste activites avec statuts et filtres.
  • Bouton AI Reco pour ouvrir recommandations.
  • Bouton Confirmer activite visible dans actions.
  • Edition inline complete (titre, seats, dates, priorite, statut).

Manager pages

Chargement competences, validation recommandations, suivi activites department, visualisation score homogene.

Employee pages

Notifications, historique, profil, et page dediee evolution competences (`/employee/skill-updates`).

5) IA/NLP: prompt, parsing, scoring

Prompt cible (court et robuste)

Le parsing nettoie puces/tirets/ponctuation, dedup, et evite faux positifs ("trouver 10 profil").

6) Workflows metier complets

Etape 1 - RH prepare activite
Creation/edit avec competences reelles de la base + niveaux cibles.
Etape 2 - IA genere classement
Top_n respecte; si IA renvoie moins, backend complete avec profils eligibles.
Etape 3 - RH confirme
Bouton envoyer/confirm; popup si places manquantes (annuler ou confirmer quand meme).
Etape 4 - Manager traite
Valide/rejette recommandations en contexte departement.
Etape 5 - Employe repond
Acceptation ou refus motive; cause enregistree et consultable RH.
Etape 6 - Post-action
Maj competences + historique + notifications + audit + event status.

7) Refus employe et actions RH

Comportement attendu

  • Cause refus toujours visible par RH.
  • RH choisit Diminuer score ou Garder score.
  • Action appliquee au moment de confirmation evenement.
  • Employe notifie de la decision RH.

Impacts techniques

  • Recalcul de rank apres penalite.
  • Log audit par action RH.
  • Notification in-app avec payload activity/recommendation.
  • Tracabilite activity_history.
