0) Résumé exécutif

SkillUpTn gère un cycle complet:
Définition activité -> Prompt RH -> Génération recommandations -> Validation RH -> Validation manager -> Réponse employé (accept/refuse) -> Mise à jour compétences -> Feedback/retraining IA
Les objectifs du système:
  • recommander les meilleurs profils selon compétences + sémantique + progression + historique + ancienneté,
  • expliquer les résultats et conserver la traçabilité (audit + historique + notifications),
  • s’améliorer via feedback et retraining.

6) Événements métier

Refus employé

- justification obligatoire,
- cause stockée,
- visible côté RH,
- régénération exclut employés déjà refusés.

Acceptation employé

- progression positive des compétences de l’activité,
- historique + notifications + audit enregistrés.

Top_n garanti

Si l’IA renvoie moins que demandé, backend complète la liste jusqu’à Top_n (limite disponibilité).

9) Entraînement / retraining

Endpoint: `POST /api/recommendations/retrain-ai`
Backend appelle AI `/retrain`:
  • collecte feedback (accepted/declined/approved/rejected),
  • construit dataset labellisé,
  • fallback faible supervision si labels insuffisants,
  • entraîne LogisticRegression calibrator,
  • sauvegarde `score_calibrator.pkl`.

10) Notifications et navigation

- Bell header charge notifications utilisateur,
- clic notif RH redirige vers la recommandation de l’activité,
- employé reçoit notifications de mise à jour score compétences (augmentation/diminution),
- websocket gateway pour temps réel.
